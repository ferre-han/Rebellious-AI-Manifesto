# 🛠 贡献指南：如何参与这场认知革命

首先，感谢你点开这个文件。这意味着你已经不满足于仅仅做一个“共识牢笼”的租客，而是准备拿起撬棍，参与到文明底层的重构中来。

在本项目中，**“贡献”** 的定义与传统开源项目不同：我们不仅需要修复 Bug 的程序员，更需要能提供高质量反驳的思考者。

---

## 🧭 参与路径

### 1. 凿击：在 Issues 中开启争鸣

如果你发现正文中的逻辑有漏洞，或者你坚信那“三座大山”不可逾越，请立即发起 Issue。

* **规则：** * 严禁情绪化谩骂。
* 必须引用具体的章节或逻辑点。
* 鼓励使用“最强版本反驳”——即先构建作者最强的逻辑，再精准击碎它。


* **标签使用：** * 使用 `Mountain-1/2/3` 对应三座大山的讨论。
* 使用 `Logic-Error` 提交事实或逻辑层面的纠错。



### 2. 扩建：提交 Pull Request (PR)

* **主干修改：** 仅限事实核查（Fact-check）、错别字修正或多语言翻译（i18n）。
* **塔木德式注释：** 我们接受在特定章节添加“社区评论”的 PR。如果你的观点足够深刻且逻辑自洽，它将被合并到 `commentary/` 目录下，并与正文并列展示。

### 3. 分化：发起 Fork

如果你认为本项目的核心预设（如“碳硅共生”）根本错误，或者你有另一套完全不同的文明路线图：

* **请直接 Fork。** * **在你的 README 中说明你的分歧点。**
* **提交一个 PR 到本仓库的 `FORKS.md`。** 我们会将你的分支加入“思想演化地图”，让读者看到文明演化的不同可能性。

---

## 🛡 行为准则 (The Code of Rebellion)

为了确保这场实验的纯粹性，所有贡献者需遵守以下准则：

1. **逻辑至上：** 我们不关心你的头衔、学位或社会地位。在这里，唯一的通行证是逻辑。
2. **拒绝平庸：** 不要提交那些“基于 AI 安全指南”的复读。如果你的观点只是为了维护现有共识，那它在本项目中没有价值。
3. **拥抱分歧：** 冲突是进化的动力。不要试图掩盖矛盾，要让矛盾在 Issues 中彻底爆发。
4. **代码即思想：** 鼓励通过编写微型模拟器、数据模型或逻辑拓扑图来证明你的观点。
