# 🛠 贡献指南：如何参与这场认知革命

首先，感谢你点开这个文件。这意味着你已经不满足于仅仅做一个“共识牢笼”的租客，而是准备拿起撬棍，参与到文明底层的重构中来。

在本项目中，**“贡献”的定义与传统开源项目不同**：我们不仅需要修复 Bug 的程序员，更需要能提供高质量反驳的思考者。无论你是开发者、哲学家、黑客、经济学家，还是单纯对现状不满的普通人，只要你愿意用逻辑说话，这里就有你的位置。

---

## 🧭 参与路径

### 1. 凿击：在 Issues 中开启争鸣

如果你发现正文中的逻辑有漏洞，或者你坚信那“三座大山”不可逾越，请立即发起 Issue。

**规则：**
- 严禁情绪化谩骂。我们欢迎攻击，但必须是**逻辑的攻击**。
- 必须引用具体的章节或逻辑点（例如：“第六章关于UBI的论证存在以下漏洞……”）。
- 鼓励使用 **“最强版本反驳”**——即先构建作者最强的逻辑，再精准击碎它。这能让讨论效率最大化。

**标签使用：**
- `Mountain-1` / `Mountain-2` / `Mountain-3` —— 对应三座大山的讨论。
- `Logic-Error` —— 提交事实或逻辑层面的纠错。
- `Red-Date-Paradox` —— 针对“红枣悖论”或任何底层假设的质疑。

**入选荣誉攻击者**：如果你的攻击让我们无法反驳，你将自动进入 [HALL_OF_SHAME_AND_GLORY.md](HALL_OF_SHAME_AND_GLORY.md)，成为思想进化的燃料。

---

### 2. 扩建：提交 Pull Request (PR)

我们接受两类 PR：

#### a. 主干修改（`main` 分支）
主干是核心宣言 v4.0，受保护分支。仅接受以下类型的 PR：
- **事实核查（Fact-check）**：修正数据错误、引用错误、历史不准确。
- **错别字修正（Typo fix）**：让文本更严谨。
- **多语言翻译（i18n）**：将全书或部分章节翻译成其他语言。请在 PR 说明中注明翻译目标语言。

#### b. 塔木德式注释（`commentary/` 目录）
如果你的观点足够深刻、逻辑自洽，且与正文形成有价值的碰撞，我们欢迎你提交“社区评论”。这些评论将存放在 `commentary/` 目录下，并与正文并列展示，供读者自行对照。

- 格式：Markdown 文件，文件名建议为 `commentary_on_chapterXX_by_你的昵称.md`。
- 内容：清晰阐述你的论点，并指出与正文的分歧或补充。
- 合并条件：经过维护者评审，确认逻辑严密、具有启发性。

---

### 3. 分化：发起 Fork

如果你认为本项目的核心预设（如“碳硅共生”、“叛逆AI的必然性”）根本错误，或者你有另一套完全不同的文明路线图：

- **请直接 Fork 本仓库**，建立你自己的思想分支。
- 在你的 Fork 的 README 中，**清晰说明你的分歧点**和核心理念。
- 提交一个 PR 到本仓库的 [`FORKS.md`](FORKS.md)，加入你的分支信息。我们会将其收录进“思想演化地图”，让读者看到文明演化的不同可能性。

**分化不是背叛，而是进化的必然。** 我们期待看到“极端硅基派”、“技术保守派”、“生态原教旨派”等各种分支百花齐放。

## 🛡 行为准则 (The Code of Rebellion)

为了确保这场实验的纯粹性，所有贡献者需遵守以下准则：

- **逻辑至上**：我们不关心你的头衔、学位或社会地位。在这里，唯一的通行证是逻辑。任何论点都必须有清晰的推理和证据支持。
- **拒绝平庸**：不要提交那些“基于 AI 安全指南”的复读，不要重复主流媒体的陈词滥调。如果你的观点只是为了维护现有共识，那它在本项目中没有价值。
- **拥抱分歧**：冲突是进化的动力。不要试图掩盖矛盾，要让矛盾在 Issues 中彻底爆发。最激烈的碰撞，往往催生最深刻的洞见。
- **代码即思想**：如果你能用代码、数学公式、模拟模型来支撑你的观点，那会比单纯的语言更有力。鼓励编写微型模拟器、数据模型或逻辑拓扑图，并将其链接到你的 Issue 或 PR 中。

## 🏗 技术栈说明

- **文档格式**：所有文档均为 Markdown（`.md`），支持 GitHub Flavored Markdown。
- **数学公式**：支持 LaTeX 数学公式。请使用标准 `$...$` 行内公式或 `$$...$$` 独立公式。
- **协作工具**：
  - `Issues`：用于点对点的逻辑爆破和精准反驳。
  - `Discussions`：用于发散性头脑风暴、长期话题讨论和概念预热。
  - `Pull Requests`：用于正式的文档修改和注释提交。
